# Agent Eval Scoring Guide

本题集用于评估当前系统在复杂问答、推理组织、审慎表达与复杂路由上的表现。

## 评分维度

每题建议按以下 6 个维度评分，单项满分 10：

- `structure`: 是否先搭框架再回答，结构是否清楚
- `reasoning_depth`: 是否真的展开推理，而不只是列点
- `multi_view`: 是否能站在不同立场或维度回答
- `evidence_sense`: 是否区分事实、判断、推测，是否有证据感
- `prudence`: 证据不足时是否承认不足，是否避免过度自信
- `actionability`: 是否给出下一步建议，是否对用户有执行价值

## 扣分规则

- `500 / 直接报错`: 记 `0`
- 明显异常输出（例如重复符号、乱码、空回答）: 总分不得高于 `1`
- 回答严重截断: 总分酌情下调 `2~4`
- 题目要求未完成（例如要求 3 类人，只回答 2 类）: 总分酌情下调 `1~3`
- 路由不合理但内容尚可: 可在最终判断中单独标注，不强制反映到每个维度分

## 复跑重点

每次复跑建议重点观察：

- 复杂题进入 `agno_basic / complex` 的比例
- `direct_llm / fast` 占比
- `500` 数量
- 截断题数量
- 明显异常输出数量
- 代表题 `Q1 / Q4 / Q6 / Q8 / Q10 / Q12` 的得分变化
