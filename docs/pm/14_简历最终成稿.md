## 版本一：通用主版本

**A1 多来源知识处理与长期复盘 AI 原型**  
面向学习复盘与知识沉淀场景的 AI 产品原型，支持视频、网页、文档和笔记的即时处理、确认保存入库，以及后续检索回查与多来源比较。

- 主导多来源 AI 知识处理原型的产品方案设计，围绕视频复盘、资料沉淀、知识回查和多链接即时比较等核心场景，抽象“高价值信息易流失、后续难回查和复用”的用户问题，并收敛产品定位与主链结构
- 提出“先处理、后保存”的产品机制，将即时问题处理与长期知识沉淀分开设计，既满足当轮任务交付，也避免低价值内容自动入库，提升知识复用质量
- 深度参与 fast / complex / async 分流、共享能力层、统一质量门、状态语义和 pending / commit / retrieval 知识链路的产品口径收敛，具备较强 AI 应用与工程实现理解
- 建立场景复盘、指标看板、竞品分析、深度方案和路由测试矩阵，形成可持续验证 AI 产品效果的评估资产，推动产品从“能调用能力”走向“能稳定交付结果”

## 版本二：偏 AI Workflow / 自动化岗位

**A1 多来源知识处理与长期复盘 AI 原型**  
面向学习复盘与知识沉淀场景的 AI Workflow 产品原型，支持资料处理、知识沉淀、检索回查和多来源即时比较。

- 负责 AI Workflow 型产品原型设计，将视频、网页、文档和笔记处理、知识沉淀、检索回查和多来源比较组织成可运行的产品工作流
- 围绕“即时处理”和“长期复用”设计双路径机制，明确当轮处理、确认保存、后续回查之间的流程边界，提升知识资产沉淀质量
- 深度参与 AI 工具协作、接口联动、状态语义定义和测试验证，具备比普通产品经理更强的 API、工具链和自动化交付理解
- 建立指标周报、sandbox 样本和测试矩阵，用于持续观察复杂任务交付、资料复用和产品主路径稳定性

## 版本三：偏 AI Agent / RAG 岗位

**A1 多来源知识处理与长期复盘 AI 原型**  
面向学习复盘与知识沉淀场景的 AI Agent 产品原型，支持即时处理与长期沉淀并存的产品主线。

- 主导一个多来源 AI Agent 产品原型的方案设计，基于视频、网页、文档和笔记场景，构建即时处理与长期沉淀并存的产品主线
- 参与 fast / complex / async 分流、Main / Middle / Answer 协作链（complex 链分层编排，非默认多 Agent 自主对话）、共享能力层和知识沉淀链路的产品口径收敛，提升复杂任务处理与统一交付能力
- 基于 RAG、工具调用和多来源材料组织，设计知识回查、即时比较和复杂任务升级机制，推动产品从基础问答走向任务级处理
- 建立路由与能力测试矩阵、指标口径和场景复盘机制，用于验证 Agent 路由稳定性、状态语义一致性和复杂任务交付质量

## 使用建议

- 如果岗位描述更偏 `AI Workflow / 自动化 / 工具提效 / AI Coding / MCP / Skills`，优先用版本二
- 如果岗位描述更偏 `Agent / RAG / 多智能体 / 工具调用 / 路由 / 复杂任务处理`，优先用版本三
- 如果岗位要求比较综合，先用版本一，再按 JD 替换关键词
