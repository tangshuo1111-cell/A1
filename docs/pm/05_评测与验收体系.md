# 评测与验收体系

目的：把「工程验收 / 路径验收 / 产品质量」三层说明在一页，便于对内沟通与对外表达。

**重要**：本文件描述的是 **代理指标与工程门禁**，不是用户调研满意度，也不是线上经营指标（留存/转化）。

---

## 三层结构

| 层级 | 回答什么 | 主要机制 | 通过标准 |
| --- | --- | --- | --- |
| **L1 工程验收** | 改完会不会回退 | CI pytest + 覆盖率 + migration/契约测试 | CI 绿；`backend` 覆盖率 ≥ 60% |
| **L2 路径验收** | 8 类请求是否走对路 | trace baseline + `05_目标运行路径与架构验收表` | lane/mode/trace 与 baseline 一致 |
| **L3 产品质量** | 复杂任务是否真正完成 | agent eval + 产品指标 v1 + 内测场景 | benchmark 不回退；代理指标可计算 |

---

## L1：单元 / 契约 / migration

| 目录 | 中文 | 何时跑 | 失败意味着什么 |
| --- | --- | --- | --- |
| `tests/unit/` | 单元测试 | 每次 CI | 模块逻辑错误 |
| `tests/integration/` | 集成测试 | 每次 CI | 跨模块主链不通 |
| `tests/smoke/` | 冒烟测试 | CI / 部署前 | 默认主路径不能跑 |
| `tests/backend/` | 后端专项 | 每次 CI | Agent / gate / API 回归 |
| `tests/acceptance/` | 验收测试 | 每次 CI | 业务终态不符合预期 |
| `tests/migration/` | 迁移契约 | 每次 CI | **架构默认路径被改坏** |
| `tests/baselines/` | 基线夹具 | 每次 CI | baseline 样本损坏 |

**契约亮点**：

- `turn_exit_gate` 单一写出口（`test_single_fact_source_contract.py`）
- `EMBEDDING_ENABLED` 统一管存+查（`test_embed_on_commit_unified.py`）
- 产品指标 canonical 字段（`test_product_metrics_v1_canonical_fields`）

## L2：trace baseline（8 类 sample）

样本目录：`docs/current/baselines/samples/`  
快照目录：`docs/current/baselines/trace_baseline_new/`  
验收表：`docs/current/03_目标运行路径与架构验收表.md`

每类请求验收四维：**lane / mode / 自治闭环 / async plane**。

**何时必须跑**：改 ingress、fast path、complex path、turn_exit_gate 后。

---

## L3：agent eval（答得好不好）

目录：`benchmarks/agent_eval/`  
指南：`benchmarks/agent_eval/scoring_guide.md`

六维打分（各 10 分）：structure / reasoning_depth / multi_view / evidence_sense / prudence / actionability。

**通过标准（相对）**：

- 代表题得分不低于 baseline JSON
- 500 / 空答 / 严重截断数量不上升
- **不**把内测场景完成率（62%–85%）作为 CI 硬门槛
