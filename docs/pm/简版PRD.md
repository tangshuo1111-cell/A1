# 简版 PRD

产品名称：
A1 多材料 AI 问答系统

## 1. 背景

当前常见 AI 问答产品多聚焦单一知识库或单轮对话场景，但真实复杂问题往往同时依赖知识库、网页、文档、视频等多种材料。现有方案在多材料接入、复杂任务分流和结果稳定性上存在明显不足，因此需要一个支持多材料处理、路由分流和统一交付的 AI 问答系统。

## 2. 目标

- 支持知识库、网页、文档、视频等多类信息输入
- 对简单问题提供快速回答
- 对复杂问题提供更稳的多阶段处理或异步任务能力
- 统一系统输出状态和结果口径
- 提升答案稳定性、可解释性和交付一致性

## 3. 用户角色

- 知识工作者
- 内容研究用户
- 需要从多来源信息中获取结论、摘要或分析的用户

## 4. 使用场景

- 用户上传文档并提问
- 用户输入网页链接并要求摘要或提取信息
- 用户提出需要结合知识库和网页信息的复杂问题
- 用户提交视频链接并要求内容提炼
- 用户提出跨材料综合问题，需要系统判断最合适路径

## 5. 功能范围

本期范围：
- 文本提问
- 文档上传
- 网页读取
- 视频处理
- 知识库检索
- 路由分流
- 多阶段生成
- 异步任务状态返回
- 统一结果输出

不在本期范围：
- 多用户权限管理
- 商业计费系统
- 企业级审计后台
- 精细化运营系统

## 6. 页面 / 流程

核心流程：
1. 用户输入问题或上传材料
2. 系统识别请求类型与复杂度
3. 系统准备检索与材料上下文
4. 请求进入快速路径、复杂路径或异步任务路径
5. 系统生成答案或任务结果
6. 结果经过质量门与统一出口后返回前端

## 7. 字段说明

核心字段建议对外统一为：
- `status`：当前请求状态
- `path`：本次请求走的主路径
- `answer`：最终回答内容
- `materials`：本次回答使用的材料摘要或来源
- `task_id`：异步任务 ID
- `pending_kind`：任务等待类型
- `trace_meta`：调试或诊断辅助信息

## 8. 交互规则

- 简单问题优先走快速路径
- 材料不足时，系统可升级为复杂路径或异步处理
- 如果最终结果未通过质量门，不直接返回低质量答案
- 对异步任务，前端需展示处理中状态而不是空白等待
- 对多材料答案，系统应尽量返回来源线索或材料依据

## 9. 异常情况

- 检索结果为空
- 网页抽取失败
- 文档解析失败
- 视频处理超时
- 质量门未通过
- 异步任务执行失败
- 外部能力不可用

异常处理原则：
- 可降级时优先 soft fail
- 不可继续时明确 hard fail
- 返回统一错误语义，不暴露内部混乱状态

## 10. 验收标准

- 默认主路径真实可跑
- 简单问题可直接返回答案
- 复杂问题能够稳定升级到复杂路径或异步路径
- 输出状态字段语义一致
- 关键出口只由统一模块写入
- 关键 benchmark 不回退
- 关键 contract / smoke / migration 测试通过

**说明**：内测场景完成率（见 `用户场景与失败复盘.md`）为观察值，**不作为**本 PRD 的 hard SLA 或 CI 门禁。产品结果验收以 `产品指标看板.md` v1 代理口径 + agent eval 相对 baseline 为准。

## 11. 数据指标

当前阶段建议指标：
- 请求成功率
- 平均响应时长
- 异步任务完成率
- 复杂问题升级率
- 质量门通过率
- 用户有效回答率
- benchmark 稳定度

后续可补：
- 留存
- 转化
- 复用率
- 单次请求成本
- 北极星指标
