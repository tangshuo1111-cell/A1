# 产品指标看板

用途：
- 定义 A1 的产品衡量口径
- 区分“请求成功”与“产品价值成立”
- 把已有周报、评测、沙箱和脚本能力整理成可对外表达的框架

## 口径说明

这份文档不是线上经营分析，也不是用户调研满意度报告。

当前指标体系主要服务于三个目标：
- 判断 A1 的核心产品价值是否成立
- 区分“资料沉淀成功”和“复杂任务真的完成”不是一回事
- 为后续周报、沙箱验证和产品迭代提供统一口径

因此，本文中的指标分为两类：
- 已落地代理指标：当前系统已具备字段或脚本口径
- 建议补充指标：产品上很重要，但当前仍需逐步接线或补充埋点

## 一、A1 应该衡量什么

A1 当前包含两条能力路径：

1. 长期沉淀路径
   - 把视频、网页、文档、笔记等内容沉淀为可长期复用的知识资产
2. 即时处理路径
   - 对多个外部来源做当轮提取、比较和基础分析

因此，A1 的指标不能只看“有没有返回答案”，而要回答三个问题：
1. 资料有没有被成功沉淀
2. 沉淀后的资料有没有被再次调用
3. 复杂或比较任务有没有真正形成可用结果

## 二、指标结构

### 1. 沉淀层指标

回答：
`内容有没有成功进入系统，成为后续可用的知识资产。`

### 2. 复用层指标

回答：
`沉淀下来的内容，后续有没有真的被找回、被追问、被继续使用。`

### 3. 分析层指标

回答：
`系统有没有把复杂任务、比较任务和深一步处理真正交付出来。`

## 三、当前最关键的产品指标

### 3.1 北极星 1：资料二次调用率

定义：
已沉淀资料中，后续至少被一次检索、问答或复盘调用的比例。

意义：
- 这是 A1 区别于普通收藏工具的关键指标
- 如果资料从未被再次调用，沉淀价值就没有真正成立

状态：
- 建议补充指标

### 3.2 北极星 2：复杂任务有效完成率

定义：
在复杂问答、多来源整合、需要升级 complex 或 async 处理的任务中，最终返回“完成且可直接使用结果”的比例。

意义：
- 衡量系统有没有从“资料可问”走向“任务可分析”

状态：
- 已落地代理指标

## 四、分层指标建议

### 4.1 沉淀层

- 资料沉淀成功率
- 视频 / 网页 / 文档解析成功率

### 4.2 复用层

- 资料二次调用率
- 基于资料的有效回答率
- 视频复盘使用率

### 4.3 分析层

- 多来源比较完成率
- Complex 有效完成率
- Partial 率
- 复杂任务升级率
- 平均材料数
- insufficiency 占比

## 五、当前最值得看的 6 个数

如果只展示一页简洁看板，建议优先展示：
1. 复杂任务有效完成率
2. Partial 率
3. 复杂任务升级率
4. 平均材料数
5. insufficiency 占比
6. 资料二次调用率

## 六、为什么这些指标比“请求成功率”更重要

A1 的问题不是“系统能不能回复”，而是：
- 内容有没有真正沉淀下来
- 后续有没有被再次调用
- 复杂任务是不是只完成了一半
- 比较任务有没有形成真实差异，而不是摘要拼接

因此，如果只看请求成功率，会掩盖很多真正影响产品价值的问题。

## 七、v1 已落地代理指标口径

版本：`product_metrics_v1`

### 7.1 复杂任务有效完成率

分子需同时满足：
- `is_complex_task == true`
- 顶层 `task_status ∈ {done, succeeded}`
- `insufficient_evidence == false`
- `quality_gate_passed == true`

分母：
同一次响应中 `is_complex_task == true` 的 turn 数。

说明：
`partial` 不计入有效完成。

### 7.2 Guardrail 字段映射

| 指标 | 计算（v1） | 字段 |
| --- | --- | --- |
| Partial 率 | `task_status == partial` / 全部 turn | 顶层 `task_status` |
| 复杂任务升级率 | `is_complex_task` / `complex_candidate` 或 ingress 标记为 complex 的 turn | `is_complex_task` + ingress trace |
| 平均材料数 | mean(`v15_retrieved_chunks_count` + 临时材料数) | `extra.v15_retrieved_chunks_count`, `extra.v15_temporary_materials_count` |
| insufficiency 占比 | `insufficient_evidence == true` / 全部 turn | `extra.insufficient_evidence` |

### 7.3 数据来源（v1）

| 优先级 | 来源 | 说明 |
| --- | --- | --- |
| 1 | PG `turn_product_metrics` | 周报主数据源 |
| 2 | Chat 响应 JSONL 导出 | `scripts/aggregate_product_metrics.py` |
| 3 | pytest / benchmark 产物 | 离线验证，不作产品看板主数据源 |
