# 协作周报规范（HTML / v1）

目的：给协作方看的**产品决策材料**（非技术日志）。  
周期：**本周 vs 上周**（建议自然周 Mon–Sun 或发版周，固定一种）。

**口径声明（页脚必带）**：

- partial **不计入**复杂任务有效完成
- eval 为离线代表题，**非全量**
- 以上为**代理指标**，非用户调研、非线上留存/转化

---

## 1. 页面结构（固定顺序）

### 1.1 标题区

| 块 | 内容 |
| --- | --- |
| 标题 | A1 产品协作周报 |
| 周期 | 本周（M/D–M/D）vs 上周 |
| 数据口径 | 链接 `产品指标看板.md` §7、`评测与验收体系.md` |

### 1.2 样本量（必补 — 否则百分比失真）

| 行 | 字段 | 说明 |
| --- | --- | --- |
| 【样本】 | `turns_total` | 总请求数 N |
| 【样本】 | `complex_task_count` | 复杂任务数（`is_complex_task=true`） |
| 【样本】 | `async_task_count` | async 样本数（`executor_profile=async` 或 async 相关 pending） |
| 【样本】 | `eval_item_count` | 离线 eval 题数（如 6） |

**协作解读**：58%→62% 在 N=8 与 N=800 下意义完全不同；样本量必须出现在所有比率之前。

### 1.3 核心结论区

| 行 | 类型 | 字段 |
| --- | --- | --- |
| 【北极星】 | 主目标 | 复杂任务有效完成率（本周 → 上周） |
| 【Guardrail】 | 护栏 1 | Partial 率 |
| 【Guardrail】 | 护栏 2 | 复杂任务升级率 |
| 【Guardrail】 | 护栏 3 | 平均材料数 |
| 【结论】 | **1 句话** | 本周整体变好/变差 + 主因（必填） |

**北极星分子条件**（v1 冻结）：`is_complex_task` 且 `task_status∈{done,succeeded}` 且 `insufficient_evidence=false` 且 `quality_gate_passed=true`。**partial 不进分子。**

### 1.4 运行与质量区

| 行 | 字段 | 类型 |
| --- | --- | --- |
| 【运行】 | 平均总耗时 | 观察 |
| 【运行】 | P95 耗时 | 观察 |
| 【运行】 | complex 完成均字数 | 观察（可选） |
| 【质量】 | quality_gate 通过率 | 观察 / 护栏 |
| 【质量】 | agent eval 均分（N 题） | 离线质量 |
| 【可选】 | async 完成率 | 观察 |
| 【可选】 | 超时率 / budget 熔断率 | 观察 |
| 【可选】 | insufficiency 占比 | **高价值护栏** |
| 【可选】 | fallback / 降级使用率 | 韧性观察（见 §3） |

### 1.5 失败结构区（必补）

| 行 | 内容 |
| --- | --- |
| 【失败 Top1】 | 原因 + 占失败/占 complex 比例 |
| 【失败 Top2】 | 同上 |
| 【失败 Top3】 | 同上 |

**建议原因 taxonomy（v1）**：

| code | 含义 | 主要字段来源 |
| --- | --- | --- |
| `insufficiency` | 材料/证据不足 | `insufficient_evidence=true` 或 `material_sufficiency` |
| `external_capability_fail` | 网页/视频/OCR/ASR 等失败 | `v16_*_error_code`、tool failures |
| `timeout_partial` | 预算/超时导致 partial | `task_status=partial` + deadline/budget trace |
| `quality_gate_block` | 质量门阻断 | `quality_gate_passed=false` |
| `upgrade_still_partial` | 已升级 complex 仍 partial | `is_complex_task` + partial |

**partial 来源拆分**（子块，可选）：insufficiency / 超时 / 外部能力 / 其他。

### 1.6 本周变化与下周动作（必补 — 推动协作）

| 行 | 写什么 |
| --- | --- |
| 【变更】 | 本周主要代码/策略/产品改动（1–2 句） |
| 【判断】 | 数字变化原因（与 guardrail、Top3 对齐） |
| 【下周】 | 优先动作 1–2 条（可验收） |

## 2. 完整示例（协作可读）

```text
本周（6/1–6/7）vs 上周（5/25–5/31）

样本：总请求 428，复杂任务 119，async 24，eval 6 题

【结论】复杂有效完成率略升，但升级率与耗时同步上升；partial 下降主要来自材料覆盖改善。

【北极星】复杂任务有效完成率  58% → 62%
【Guardrail】Partial 率           18% → 15%
【Guardrail】复杂升级率           41% → 43%
【Guardrail】平均材料数           2.1 → 2.4

【运行】平均总耗时               8.2s → 9.1s
【运行】P95 耗时                 24s → 26s
【质量】quality_gate 通过率      88% → 90%
【质量】agent eval 均分（6题）   6.2 → 6.4（baseline 6.0）

【失败 Top1】insufficiency           34%
【失败 Top2】外部能力失败           27%
【失败 Top3】超时转 partial          19%

【变更】复杂题质量门略收紧；前端增加材料依据摘要。
【判断】有效完成率上升主要来自复杂任务材料覆盖改善；耗时上升与 complex 升级率增加相关。
【下周】优先压缩 complex 路径耗时；降低 insufficiency 导致的 partial。

说明：partial 不计入有效完成；eval 为离线代表题，非全量；以上为代理指标，非用户调研。
```

## 3. 字段 → 数据来源（实现对照）

| 周报字段 | 状态 | 数据来源 |
| --- | --- | --- |
| 样本量 / 四指标 | ✅ | PG `turn_product_metrics` + `scripts/report_product_metrics.py` |
| 耗时 mean/P95 | ✅ | `timing_total_ms` 列 |
| 输出字数 | ✅ | `answer_char_count` 列 |
| quality_gate / insufficiency | ✅ | exit gate → PG |
| 失败 Top3 | ✅ | `failure_reason_code` 聚合 |
| fallback 使用率 | ⬜ Phase 2.1 | `v16_*_error_code` |
| eval 均分 | 手动 | `benchmarks/agent_eval/` |
| 【变更】【判断】【下周】 | 人工 | `weekly_*_notes.md` |

**聚合脚本（已落地）**：

- 写入：`turn_exit_gate` → PG（soft fail）
- 报告：`py -3.12 scripts/report_product_metrics.py --days 7 --html`
- 产物：`_local/reports/metrics/weekly_*.json` + `.html`

## 4. HTML 版式建议（简洁）

1. 顶部：周期 + 样本量（大字）
2. 一块「一句话结论」（高亮）
3. 表格：指标 | 本周 | 上周 | Δ
4. 失败 Top3：横向条形或三行百分比
5. 灰底区块：变更 / 判断 / 下周
6. 页脚：口径免责声明

不必图表库；静态 HTML + 内联 CSS 即可托管在 `_local/metrics/` 或 Release 附件。
