后续目录调整、运行时迁移、命名治理、测试治理都必须回扣这一目标，不再以“统一重主链”作为默认终局结构。

## 冻结结论

### 总体结构

外层负责：

- `Semantic Router`
- `Fast Lane / Complex Lane` 分流
- `Shared Capability Plane`
- `Async Control Plane`

内层负责：

- `Three-Agent Autonomous Core`
- `MainAgent -> MiddleAgent -> AnswerAgent`

### 结构原则

1. 简单请求优先走 `Fast Lane`，避免默认进入完整重链。
2. 复杂请求进入 `Complex Lane`，由三强自治核心承担规划、编排、验证与签字。
3. 视频、文档、网页、知识库能力统一沉到共享能力平面，而不是继续散落在单个主链文件中。
4. 长任务和重任务统一进入异步控制平面，而不是继续由同步主链硬扛。

## 术语表

### `Semantic Router`

入口层语义路由器。职责是根据规则、轻量分类器和必要时的升级判断，决定请求走哪条 `lane`、采用哪种 `mode`。它不等于 `MainAgent`，也不取代三强自治核心。

### `Fast Lane`

快速同步通道。目标是在最短首答路径上完成明确、轻量、高置信请求。允许使用轻量编排，但禁止进入完整 `revise / retry / reflect` 闭环。

### `Complex Lane`

复杂同步通道。用于低置信、跨能力、需补证、需多步策略控制的请求。复杂请求在这里进入三强自治核心闭环。

### `Shared Capability Plane`

共享能力平面。承接视频、文档、网页、知识库和通用执行能力，统一作为 Middle 或外层快链的可复用能力层。

### `Async Control Plane`

异步控制平面。统一承载长视频、多视频、大文档、多网页、长耗时研究任务的后台执行、状态查询、结果提交与恢复。

### `Three-Agent Autonomous Core`

三强自治核心，指 `MainAgent / MiddleAgent / AnswerAgent` 的稳定闭环协作内核。它只在复杂模式下完整发挥作用，不负责 provider 细节和底层执行实现。

## 角色冻结

### `MainAgent = Planner / Router / Manager`

- 负责复杂模式下的规划、复杂 lane 内策略控制、复杂升级与回路管理
- 不取代入口层 `Semantic Router`
- 不直接承载底层工具与 provider 细节

### `MiddleAgent = Capability Orchestrator`

- 负责能力编排、材料收集、重试、fallback、同步/异步切换
- 不负责最终签字
- 不应继续吸收底层重处理实现

### `AnswerAgent = Critic / Verifier / Sign-off`

- 负责批判、验证、是否补料、最终签字
- 输出最终回答及签字相关链路信息
- 不负责底层抓取、OCR、ASR、队列实现

## 平台层定义冻结

### 外层平台层

- `application/`：入口、模式控制、路由与平台编排
- `services/`：共享能力与执行策略平面
- `tasks/`：异步控制平面
- `workers/`：后台执行平面
- `tools/`：工具适配层 / provider wrapper

### 内层自治核心

- `agents/`：三强自治核心唯一归属目录
