# 02 目录规则

## 顶层目录

- `backend/`：后端源码
- `frontend/`：前端源码
- `tests/`：测试
- `data/`：可提交演示样本
- `docs/`：项目事实文件
- `scripts/`：项目级操作脚本
- `_local/`：本地运行产物

## backend 规则

### tools

`backend/tools/` 只负责“拿内容”。

例如：

- 抓网页
- 拉视频字幕
- 读文件
- 调搜索

它不直接决定是否入库。

### knowledge

`backend/knowledge/` 只负责“把内容变成知识材料，并进入有状态知识链”。

包括：

- parse
- chunk
- pending
- commit
- index
- search

### providers

`backend/providers/` 只包外部 API / 模型供应商接缝。

不承载业务编排。

### storage

`backend/storage/` 只做存储访问，不做业务判断。

## scripts 规则

`scripts/` 放根目录，不放进 `backend/`。

这里放的是：

- 启动
- smoke
- 清缓存
- 初始化 `_local`
- 环境检查
- 打包
- 审计

## _local 规则

`_local/` 是本地运行区，不提交、不打包。

包含：

- `data/`
- `cache/`
- `logs/`
- `temp/`
- `uploads/`
- `outputs/`
- `reports/`

## tests 规则

新测试默认不再放在 `tests/` 根目录。

优先归入：

- `tests/backend/`
- `tests/integration/`
- `tests/smoke/`
- `tests/acceptance/`

已不再跟随主链、仅作考古的历史用例放入 `tests/legacy/`（`pytest` 默认 `testpaths` 不收集该目录）。

## 工程治理规则

迁移和治理期间遵守这些红线：

- 不新增大块头单文件，已有大文件继续按 helper / service / builder 拆分。
- 不随意新增顶层目录，新能力优先放入现有 `backend/`、`frontend/`、`tests/`、`scripts/`、`data/`、`docs/`、`_local/`。
- 默认主链的主判断权留在 `MainAgent`、`MiddleAgent`、`AnswerAgent`，外层只做编排和适配。
- 高风险重构前保留可回退点，重构后必须跑固定回归。
- 新增依赖先写入 `pyproject.toml`，再用与 CI 相同的 **Python 3.12** 执行 `python -m piptools compile pyproject.toml --extra dev -o requirements.lock --strip-extras` 更新 `requirements.lock`。**事实源**：`pyproject.toml` + `requirements.lock`（已删除易漂移的 `requirements.txt`）。

## 当前文档规则

`docs/current/` 只保留当前仍承担事实源职责的文档，不再把施工记录、阶段总结、重复验收稿长期放在 current。

- 工程治理说明归入 `docs/current/02_目录规则.md`。
- 环境与数据库（PostgreSQL、`DATABASE_URL`、`docker-compose`、`.env`）见 `docs/current/08_环境变量与数据库.md`。
- 默认路由、材料流与质量门控规则见 `docs/current/17_默认路由_材料流与质量门控规则.md`。
- 契约定义统一归入 `docs/current/contracts/`。
- 回归基线统一归入 `docs/current/baselines/`。
- 迁移台账统一归入 `docs/current/migration/`。
- 评测补强材料只保留仍被 benchmark 脚本消费的 `20`~`25`。
- `docs/evidence/` 只保留说明入口，不再存放仓库内大包。
- 根目录只保留当前入口文档 `README.md`。
