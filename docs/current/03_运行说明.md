# 03 运行说明

这份文档不是泛述，而是**本机真实可执行的启动步骤**。  
目标：把数据库、后端、前端整套跑起来，并且让前端通过 **Next.js 代理** 访问后端。

数据库连接、PostgreSQL 说明、`.env` 约定补充见 `docs/current/08_环境变量与数据库.md`。

## 0. 先确认当前项目的真实配置口径

本项目当前实际情况如下：

1. 后端本地配置文件在：`项目代码/.env`
2. 前端本地配置文件在：`项目代码/frontend/.env.local`
3. 这两个文件都已经被 `项目代码/.gitignore` 忽略，不会提交到 Git
4. 视频 cookies 的**正确本地位置**在：`项目代码/_local/data/cookies/video_cookies.txt`
5. 前端默认应该走 Next 的 `/api-proxy/*`，**不要让浏览器直接连后端**
6. 本机启动顺序应该是：
   1. 先起 Docker 里的 PostgreSQL 和 Redis
   2. 再起后端
   3. 再起前端
7. `data/` 下旧 sqlite 已经归档到：`项目代码/_local/data/backup_sqlite/`
8. `D:\1\env..txt` 不是项目自动读取的配置源；当前项目真正会读取的是 `项目代码/.env`

## 1. 你要改的配置文件在哪

### 后端配置

后端运行时配置放在：

```text
项目代码/.env
```

这里通常放：

- 数据库连接串 `DATABASE_URL`
- 主大模型 key `LLM_API_KEY`
- 大模型地址 `LLM_BASE_URL`
- 模型名 `LLM_MODEL`
- 视频无字幕时的云 ASR 配置：`ASR_ENABLED`、`ASR_PROVIDER`、`ASR_MODEL`、`ASR_BASE_URL`
- 搜索、OCR、ASR、腾讯云等工具相关 key

这些都属于**本地敏感配置**，当前项目的正确放法就是继续放在：

```text
项目代码/.env
```

不要把这些 key 写进：

- `docs/`
- `data/`
- `README.md`
- 前端源码

当前项目**不会自动读取**下面这个文件：

```text
D:\1\env..txt
```

所以即使你把 Tavily、腾讯 OCR/ASR key 写在那里，项目默认也不会生效。  
要让项目真正用上，必须把它们写入：

```text
项目代码/.env
```

### 前端配置

前端本地开发配置放在：

```text
项目代码/frontend/.env.local
```

这里当前最关键的是：

- `BACKEND_URL=http://127.0.0.1:8000`

它的作用不是让浏览器直连后端，而是告诉 **Next 开发服务器**：  
把 `/api-proxy/*` 请求转发到哪个后端地址。

如果后端以后开启了 `API_BEARER_TOKEN`，前端对应放法仍然是写在：

```text
项目代码/frontend/.env.local
```

例如：

```text
NEXT_PUBLIC_API_BEARER_TOKEN=和后端相同的token
```

不要把 Bearer token 写进：

- 前端组件源码
- `docs/`
- `data/`

### 视频 cookies 的正确位置

视频 cookies 不属于“可提交样本”，而属于**本地运行态敏感文件**。

当前项目的正确位置是：

```text
项目代码/_local/data/cookies/video_cookies.txt
```

这次已经把旧位置：

```text
项目代码/data/cookies/video_cookies.txt
```

移动到了正确位置。  
原因是 `data/` 是可提交演示样本区，而登录态 cookies 应该放进 `_local/` 这种本地运行区。

### 旧 sqlite 的正确位置

当前项目业务主路径是 PostgreSQL，不再把 sqlite 当作默认运行数据库。  
如果历史上留下了 sqlite，本地保留位置应当是：

```text
项目代码/_local/data/
```

如果是这次从 `data/` 清理出来、又不想直接删除的旧 sqlite，则归档到：

```text
项目代码/_local/data/backup_sqlite/
```

不要再把这些 sqlite 放回：

```text
项目代码/data/
```

### 为什么这些文件不会进 Git

因为 `项目代码/.gitignore` 已经忽略了：

- `.env`
- `**/.env`
- `.env.local`
- `frontend/.env.local`
- `_local/`
- `**/data/cookies/*`
- `**/video_cookies.txt`

所以这些本地 key 和本地地址不会被提交。

## 2. 浏览器为什么不要直连后端

你现在项目里的真实配置是：

- 前端代码默认请求 `/api-proxy/*`
- `frontend/app/api-proxy/[...path]/route.ts` 会把 `/api-proxy/*` 转发到 `BACKEND_URL`
- `frontend/.env.local` 当前就是给这个代理目标用的

所以本机联调时，正确路径是：

```text
浏览器
-> http://127.0.0.1:3000/api-proxy/...
-> Next.js 开发服务器
-> 转发到 http://127.0.0.1:8000/...
-> FastAPI 后端
```

本机开发**不要**主动设置：

```text
NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000
```

因为这个变量一旦设置，浏览器就会绕过 Next，直接访问后端。  
而你当前要求是：**浏览器不能直接连后端，要走 Next 代理。**

## 3. 先检查本地配置文件

### 3.1 检查后端 `.env`

先进入项目根目录：

```powershell
Set-Location "D:\1\Rag_and_Agent_repo_staging\项目代码"
```

如果没有 `.env`，先复制模板：

```powershell
Copy-Item .env.example .env
```

然后打开 `项目代码/.env`，重点确认这些项目已经存在并且不是空的：

- `APP_ENV=dev`
- `LIGHT_MAQA_FAKE_LLM=0`
- `DATABASE_URL=postgresql://light_maqa:...@127.0.0.1:5432/light_maqa`
- `LLM_API_KEY=...`
- `LLM_BASE_URL=...`
- `LLM_MODEL=...`
- `ASR_ENABLED=1`
- `ASR_PROVIDER=siliconflow`
- `ASR_MODEL=FunAudioLLM/SenseVoiceSmall`
- `ASR_BASE_URL=https://api.siliconflow.cn/v1`
- `V16_ENABLE_EXTERNAL_PROCESSING=1`
- `V16_ENABLE_PAID_ASR=1`
- `V16_ENABLE_PAID_OCR=1`
- `V16_ASR_PROVIDER=siliconflow`
- `V16_OCR_PROVIDER=tencent`

如果你要走外部工具，也继续确认这些对应 key：

- 网页搜索：
  - `V16_WEB_SEARCH_PROVIDER=tavily`
  - `V16_WEB_SEARCH_API_KEY`
- 腾讯 OCR / ASR：
  - `V16_ENABLE_EXTERNAL_PROCESSING=1`
  - `V16_ENABLE_PAID_ASR=1`
  - `V16_ENABLE_PAID_OCR=1`
  - `V16_ASR_PROVIDER=siliconflow`
  - `V16_OCR_PROVIDER=tencent`
  - `V16_TENCENT_APPID`
  - `V16_TENCENT_SECRET_ID`
  - `V16_TENCENT_SECRET_KEY`
  - `V16_TENCENT_REGION`

说明：

- `LIGHT_MAQA_FAKE_LLM=0` 表示不要走假回答
- 如果这里写成 `1`，回答会变成测试回答，不是真实模型
- `LLM key` 和大多数工具 key 的正确位置就是 `项目代码/.env`
- **不要**把这些 key 拆到 `data/` 或 `_local/` 里零散存放
- 当前视频链的无字幕兜底 ASR，默认先走 SiliconFlow/OpenAI 兼容 ASR；若主通道调用失败，系统可再兜底尝试腾讯 ASR

### 3.2 检查前端 `.env.local`

打开：

```text
项目代码/frontend/.env.local
```

确认至少有这一行：

```text
BACKEND_URL=http://127.0.0.1:8000
```

并且**不要启用**浏览器直连变量。也就是下面这行应该保持不写，或继续注释：

```text
# NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000
```

### 3.3 检查视频 cookies 文件

去看这个位置：

```text
项目代码/_local/data/cookies/video_cookies.txt
```

如果文件存在，说明当前本地 cookies 已经放在正确位置。  
如果你在旧位置 `data/cookies/` 找到同名文件，说明它放错了，应该移动到 `_local/data/cookies/`。

### 3.4 检查旧 sqlite 是否还留在 data/

现在正确口径是：

- 运行中的本地 sqlite：`项目代码/_local/data/`
- 历史归档 sqlite：`项目代码/_local/data/backup_sqlite/`

所以 `项目代码/data/` 下不应该再留：

- `conversation.sqlite`
- `knowledge.sqlite`
- `runtime.sqlite`

## 4. 启动前先准备依赖

在项目根目录执行：

```powershell
Set-Location "D:\1\Rag_and_Agent_repo_staging\项目代码"
python scripts/init_local_dirs.py
python scripts/check_env.py
```

你应该看到：

- 关键目录存在
- `LLM_API_KEY=SET` 或 `OPENAI_API_KEY=SET`

如果这里显示 LLM key 是 `EMPTY`，说明后端还读不到真实 key。

## 5. 第一步：启动数据库 Docker

这一步先把 PostgreSQL 和 Redis 起起来。

在 `项目代码/` 目录执行：

```powershell
docker compose -p rag_agent up -d postgres redis
```

然后查看状态：

```powershell
docker compose -p rag_agent ps
```

通过标准：

- `postgres` 显示 `healthy`
- `redis` 显示 `healthy` 或 `running`

如果你想看 PostgreSQL 日志：

```powershell
docker compose -p rag_agent logs postgres --tail=50
```

## 6. 第二步：启动后端

仍然在 `项目代码/` 目录执行：

```powershell
python scripts/run_dev.py --backend
```

这条命令会：

- 用 `PYTHONPATH=backend`
- 启动 `uvicorn api.main:app`
- 默认监听 `127.0.0.1:8000`

### 后端启动成功怎么判断

新开一个 PowerShell 窗口，仍进入 `项目代码/`，执行：

```powershell
curl http://127.0.0.1:8000/health
```

看到下面这类结果算通过：

- 返回 JSON
- `status` 是 `ok` 或 `degraded`
- `checks.postgresql.status` 是 `ok`

### 再验证后端是不是走真实 LLM

继续执行：

```powershell
curl -X POST http://127.0.0.1:8000/chat/agno `
  -H "Content-Type: application/json" `
  -d "{\"message\":\"你好，请简短介绍你自己\"}"
```

通过标准：

- 返回 200
- 有 `answer` 或等价回答字段
- 回答内容**不是** `测试回答：...`

如果出现下面情况，代表什么问题：

- 返回 `测试回答：...`
  - 说明你还在 fake 模式
- 返回 `NO_LLM_KEY`
  - 说明 `.env` 里的 LLM key 没被正确读到
- `/health` 不通
  - 先检查 Docker 数据库是否真的起来了

## 7. 第三步：启动前端

再开一个 PowerShell 窗口，进入：

```powershell
Set-Location "D:\1\Rag_and_Agent_repo_staging\项目代码\frontend"
```

第一次运行先装依赖：

```powershell
npm install
```

启动前端：

```powershell
npm run dev
```

默认地址：

```text
http://127.0.0.1:3000
```

## 8. 第四步：验证前端确实走的是 Next 代理

浏览器打开：

```text
http://127.0.0.1:3000
```

然后：

1. 打开聊天页面
2. 输入一句话，例如：`你好`
3. 打开浏览器开发者工具 `Network`
4. 看请求地址

通过标准：

- 请求应打到：

```text
POST /api-proxy/chat/agno
```

- 而不是：

```text
http://127.0.0.1:8000/chat/agno
```

这说明浏览器没有直连后端，而是先走 Next，再由 Next 转发。

## 9. 如果你要走真实外部工具，要检查哪些 key

下面不是“都必须配”，而是“你要用哪项能力，就配哪项”。

### 9.1 主聊天 LLM

至少要有：

- `LLM_API_KEY`
- `LLM_BASE_URL`
- `LLM_MODEL`

### 9.2 网页搜索

如果你要走搜索能力，确认：

- `V16_WEB_SEARCH_API_KEY`

### 9.3 腾讯 OCR / ASR

如果你要走腾讯相关能力，确认：

- `V16_ENABLE_EXTERNAL_PROCESSING=1`
- `V16_ENABLE_PAID_ASR=1`
- `V16_ENABLE_PAID_OCR=1`
- `V16_ASR_PROVIDER=siliconflow`
- `V16_OCR_PROVIDER=tencent`
- `V16_TENCENT_APPID`
- `V16_TENCENT_SECRET_ID`
- `V16_TENCENT_SECRET_KEY`
- `V16_TENCENT_REGION`

### 9.4 视频相关

视频网页抓取和字幕链不一定每次都要额外 key，但如果涉及受限页面、登录态页面，通常还要 cookies。

当前项目里，视频 cookies 的正确本地路径是：

```text
项目代码/_local/data/cookies/video_cookies.txt
```

不要再去 `data/cookies/` 下找真实 cookies。

### 9.5 视频无字幕时的 ASR 兜底

当前项目的目标口径是：

1. 先抓视频字幕
2. 如果没有字幕，自动提取音频
3. 再走云端 ASR 转文字

要让这条链路一开始就可用，`.env` 里至少要有：

```text
ASR_ENABLED=1
ASR_PROVIDER=siliconflow
ASR_MODEL=FunAudioLLM/SenseVoiceSmall
ASR_BASE_URL=https://api.siliconflow.cn/v1
V16_ENABLE_EXTERNAL_PROCESSING=1
V16_ENABLE_PAID_ASR=1
V16_ASR_PROVIDER=siliconflow
```

并且要有：

```text
LLM_API_KEY=你的可用 key
```

当前项目还支持把腾讯 ASR 作为兜底，因此下面这组也建议保留：

```text
V16_TENCENT_APPID=...
V16_TENCENT_SECRET_ID=...
V16_TENCENT_SECRET_KEY=...
V16_TENCENT_REGION=ap-guangzhou
```

如果你还要让腾讯 OCR 一开始也可用，继续确认：

```text
V16_ENABLE_PAID_OCR=1
V16_OCR_PROVIDER=tencent
V16_TENCENT_SECRET_ID=...
V16_TENCENT_SECRET_KEY=...
V16_TENCENT_REGION=ap-guangzhou
```

## 11. 常见问题

### 前端能打开，但发消息失败

先查三件事：

1. 后端是不是已经在 `127.0.0.1:8000` 启动
2. `frontend/.env.local` 里的 `BACKEND_URL` 是不是 `http://127.0.0.1:8000`
3. 浏览器有没有错误地直连后端

### 返回的是测试回答

检查 `项目代码/.env`：

- `LIGHT_MAQA_FAKE_LLM` 应该是 `0`

### 后端报数据库连接错误

先执行：

```powershell
docker compose -p rag_agent ps
```

看 PostgreSQL 是否 healthy。

### 有 key 但还是报没配置

先执行：

```powershell
python scripts/check_env.py
```

如果它仍显示 `LLM_API_KEY=EMPTY`，说明当前后端进程没有读到你的 `.env`。

同时请注意：

- `D:\1\env..txt` 不是项目自动读取的配置文件
- 真正会生效的是 `项目代码/.env`

### 视频链接抓取失败，怀疑是 cookies 没配上

先检查这个文件是否存在：

```text
项目代码/_local/data/cookies/video_cookies.txt
```

如果文件不存在，说明当前本地还没有可用的视频 cookies。  
如果你在 `data/cookies/` 找到旧文件，那是旧位置，应该以 `_local/data/cookies/` 为准。

### 视频没有字幕，系统却提示 ASR 问题

先按这个顺序排查：

1. 确认 `.env` 里已有：
   - `ASR_ENABLED=1`
   - `ASR_PROVIDER=siliconflow`
   - `ASR_MODEL=FunAudioLLM/SenseVoiceSmall`
   - `ASR_BASE_URL=https://api.siliconflow.cn/v1`
2. 确认 `.env` 里有可用的 `LLM_API_KEY`
3. 如果主 ASR 通道失败，系统会尝试使用腾讯 ASR 兜底；因此也建议保留：
   - `V16_ENABLE_EXTERNAL_PROCESSING=1`
   - `V16_ENABLE_PAID_ASR=1`
   - `V16_ASR_PROVIDER=siliconflow`
   - `V16_TENCENT_APPID`
   - `V16_TENCENT_SECRET_ID`
   - `V16_TENCENT_SECRET_KEY`
   - `V16_TENCENT_REGION`

### 页面里怎么看这次到底慢了多少秒

现在前端会在两个位置显示更明显的耗时：

1. 正在生成时：
   - 显示 `正在整理回复`
   - 同时显示 `已等待 X.X 秒`
2. 助手消息返回后：
   - 在消息底部显示 `本轮耗时 X.X 秒`

这样你就能直接看见这一轮到底慢了多少秒，而不是只能凭感觉等。
