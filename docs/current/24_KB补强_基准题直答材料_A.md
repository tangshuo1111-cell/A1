# 24 KB补强：基准题直答材料 A

## 用途

本文件专门为 KB benchmark 中的 `KBX1 ~ KBX5` 提供直答型知识块。

目标不是取代架构文档，而是让知识库在自然语言检索阶段，能更直接命中与题目表述高度接近的材料。

---

## KBX1

### 问题

请结合知识库中的协作施工计划、项目评估与上线建议，判断当前系统最影响真实用户体验的三个问题，并按优先级排序。

### 直答材料

当前系统最影响真实用户体验的三个问题，按优先级排序可概括为：

1. **复杂题升级与完整交付不稳定**  
   这是第一优先级，因为用户最直接感受到的是复杂题“有时答得很深，有时像只做了半步”。如果复杂题没有稳定进入 Agent 主链，或进入 complex 后仍 frequently partial，体感会最差。

2. **KB 检索证据覆盖度与厚度不足**  
   这是第二优先级，因为知识题如果只能拿到 `0/1` chunk，系统即使结构正确，也容易显得证据薄、结论不稳，最终又会退到 web fallback。

3. **专项重处理链的同步交付稳定性仍不够**  
   这是第三优先级。视频、文档、重资料链虽然主路已收口，但复杂或长耗时路径仍可能 partial，说明“主链接上了”还没有完全变成“交付稳定”。

## KBX2

### 问题

请基于知识库说明：当前系统为什么会出现“复杂题没有稳定进入 Agent 主链”的现象？请从路由、材料流、结果闭环三方面分析。

### 直答材料

当前系统出现“复杂题没有稳定进入 Agent 主链”的原因，可以从三方面分析：

1. **路由层面**  
   系统默认先 fast 是合理的，但如果 `complex candidate` 规则和 `quality_gate` 过于宽松，就会出现复杂题被 fast 吞掉的问题。复杂题并不是“有个答案”就够，而应看有没有真正完成比较、取舍、分情况分析。

2. **材料流层面**  
   如果共享检索、`kb_sufficiency`、temporary / pending / committed 三层材料没有被同一套协议消费，就会出现 fast、Middle、complex 使用不同证据集的情况，导致复杂题即使升级，也不稳定。

3. **结果闭环层面**  
   复杂题进入 Agent 并不等于复杂题就已经完整交付。当前还存在 round0 可以升级，但最终仍 partial 的情况。这说明结果闭环还受制于预算、证据厚度和二轮 refine 质量。

简化结论：

**当前问题不是“没有 Agent 主链”，而是“复杂题升级门槛、材料流一致性和结果闭环质量还不够稳”。**

## KBX3

### 问题

请结合知识库中的协作施工计划、Agent 评测基准和测试质量审计，判断当前系统的最大质量风险到底是“路由错误”“回答截断”还是“链路不闭环”，并说明为什么。

### 直答材料

当前系统的最大质量风险，更接近：

## 路由错误

原因：

1. **链路不闭环问题已明显改善**  
   `multisource_compare` 已并入统一二轮 gate，approval 与 commit 闭环更清楚，material trace 也更一致，所以“链路不闭环”仍是风险，但不再是第一风险。

2. **回答截断更多是表层结果**  
   partial 或截断通常是更深层问题的结果，例如复杂题没有稳定升级、材料不足、complex 预算内无法完整组织，所以“回答截断”更像现象层。

3. **路由错误会放大其他问题**  
   本该进 Agent 的复杂题如果被留在 fast，就会同时表现出浅答、结构不足、比较不充分等一系列问题，因此它是更上游的质量风险。

## KBX4

### 问题

请基于知识库，对比当前系统中的 Agent 主链、capability 主链、async task 主链，分别说明它们最适合解决什么问题、最不适合解决什么问题。

### 直答材料

### Agent 主链

最适合：

- 复杂分析
- 多维判断
- 比较与取舍
- 路线图与整改建议
- 基于多份材料做结论组织

最不适合：

- 长后台重处理
- 视频下载、切段、ASR、OCR 这类工程型任务
- 单事实或单术语快答

### capability 主链

最适合：

- 视频、网页、文档、知识检索等专项能力
- 工具执行
- 把重资料变成 temporary / pending / committed 材料

最不适合：

- 最终复杂比较
- 多步推理与取舍
- 复杂题收口表达

### async task 主链

最适合：

- 长视频
- 大文档
- 明确超出同步预算的重任务
- 适合先 pending / 后台完成的处理

最不适合：

- 本可同步完成的复杂知识判断
- 需要即时结论和比较的题

## KBX5

### 问题

请结合知识库判断：如果现在只能优先修一个方向，应该优先修视频链、KB 检索链，还是复杂题进入 Agent 主链的路由？请给出你的推荐和取舍理由。

### 直答材料

如果现在只能优先修一个方向，最推荐优先修：

## 复杂题进入 Agent 主链的路由与升级质量门控

理由：

1. **这是当前最大的放大器问题**  
   如果复杂题本该进 Agent 却没进，KB 再强、视频链再通，最终复杂结论依然可能偏浅。

2. **它最直接影响复杂题用户体验**  
   用户不会先感知内部有几条链，而是先感知“复杂题到底答没答到位”。

3. **它能放大现有投入价值**  
   你已经做了主链接线、shared retrieval、quality_gate、approval_gate、material trace，如果复杂题升级不稳，这些建设的价值就释放不出来。

取舍上：

- 视频链重要，但更偏专项能力
- KB 检索链重要，但更偏证据厚度
- 复杂题路由是最上游的价值放大器，因此应优先
