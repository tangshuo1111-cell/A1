# RAG 检索：关键词、语义与 Hybrid 混合检索说明

## 什么是 RAG

RAG（Retrieval-Augmented Generation，检索增强生成）指：在生成回答前，先从知识库检索与问题相关的证据片段（chunk），再把这些证据连同问题一起交给大模型作答。它把「事实来源」与「语言组织」解耦，降低幻觉、提升可溯源性。

## 三种检索策略

本系统的 RAG 检索支持三种策略，并由 `auto` 自动选路：

1. **keyword（关键词 / FTS）**：基于全文检索（Full-Text Search），按词项匹配打分。优点是对专有名词、代号、精确字符串命中稳；缺点是对同义改写、概念型问题召回弱。
2. **semantic（语义 / 向量）**：把 query 和 chunk 编码成向量（embedding），按余弦相似度召回。优点是能跨措辞理解语义、对概念型问题友好；缺点是对精确字符串、罕见专名可能漂移。
3. **hybrid（混合）**：同时跑 FTS 与语义两路，再按权重融合分数。这是召回质量与稳健性的折中，也是有 embedding 数据时的默认选择。

## Hybrid 的权重融合

Hybrid 用一个权重 `alpha` 平衡两路分数：

- `alpha` 为 **FTS（关键词）权重**，`1 - alpha` 为**语义权重**。
- 系统默认 `alpha = 0.45`，即语义权重 0.55 略高于关键词，兼顾概念召回与精确匹配。
- 融合分数大致为：`score = alpha * fts_score + (1 - alpha) * semantic_score`（各路分数先归一化）。

## keyword 与 semantic 如何配合

- **关键词负责"对得准"**：保证含有精确词项（代号、API 名、错误码）的 chunk 不被语义漂移漏掉。
- **语义负责"找得全"**：把换了说法、但概念一致的 chunk 召回进来。
- **Hybrid 让两者互补**：先各自召回 top-k，再融合排序；关键词强命中能把语义错漏拉回，语义能补关键词的同义召回盲区。
- **证据不足时要显式说明**：当两路融合后命中分数都低（kb_hits 少或 top_score 低），应判定 KB 证据不足（evidence_tier 弱），并在回答中明确说明不确定性，而不是强行作答。

## auto 选路规则

- 当知识库**有 embedding 数据（≥1 条）**时，`auto` 选 **hybrid**（FTS 权重 alpha=0.45，语义权重 0.55），跳过纯 keyword 与纯 semantic。
- 当**没有 embedding 数据**时，`auto` 退化为 **keyword（仅 FTS）**。
- 这样保证：向量未就绪时仍可检索，向量就绪后自动升级到混合检索。
