## 1. 项目名称

---

## 2. 项目当前目标

基于 `README.md`、`AGENTS.md`、`docs/current/` 与代码结构推断，本项目是一个面向**学习复盘、知识沉淀、多来源信息处理**的多 Agent AI 原型系统，核心验证：

1. **即时处理 + 长期沉淀双路径**：先把网页/视频/文档处理成当轮可用结果，用户明确确认后再 `commit` 入库，而非自动入库。
2. **Fast / Complex / Async 分流**：简单问答走 Fast；需要比较、判断、组织材料的任务走 Complex（Main → Middle → Answer）；少数重任务（动态网页、长视频 ASR、OCR 等）走 Async 后台。
3. **多来源能力**：网页抓取、文档解析、视频字幕/ASR、知识库检索（RAG hybrid/keyword 自动降级）。
4. **可验证的工程化**：质量门、材料 trace、pending/commit/retrieve 语义、测试矩阵与 CI 门禁。

当前版本线：**V16**（Tool 集成与真实资料处理）、**V17**（三 Agent 架构与多来源协商）。

## 11. 当前能看出来的运行方式

### 依赖准备

```powershell
python -m pip install -r requirements.lock
docker compose up -d postgres redis   # 或 docker compose up 全栈
Copy-Item .env.example .env           # 填写 LLM/ASR 等 key（本地 .env 不入库）
```

### 后端

```powershell
python scripts/run_dev.py --backend
# 或
python scripts/run_dev.py --backend
```

### 前端

```powershell
cd frontend
npm install
Copy-Item .env.example .env.local
npm run dev
```

前端默认通过 Next.js `/api-proxy/*` 访问后端，避免浏览器直连 CORS 问题。

- Playwright Chromium（动态网页）
- ffmpeg（视频 ASR 链）
- Tesseract（仅 local OCR fallback 时需要）

## 12. 当前能看出来的主要问题或不确定点

1. **工作区有未提交改动**：git status 显示 ASR 后台执行、complex session writeback、task dispatcher 等文件正在开发中，审查包反映的是**当前工作树快照**，未必与远端 main 一致。
2. **路径口径**：README 中仍出现 `项目代码/` 作为目录前缀描述，与实际仓库根 `D:\1\A1_publish` 命名不完全一致。
3. **Complex 链复杂度高**：Fast/Complex/Async 与三 Agent、二轮补材、质量门交织，新人需按 `docs/current/04_默认路由_材料流与质量门控规则.md` 对照代码，否则易误解为“所有请求都走三 Agent”。
4. **Async / 后台任务**：Redis + task plane 与 ASR mid-background worker 相关改动较多，边界与退出语义需结合 `docs/history/current/平台图+三强自治核心架构迁移执行计划.md` 验收。
5. **Embedding 与检索降级**：默认 `RETRIEVAL_MODE=auto` + `EMBEDDING_ENABLED=1`，实际路径依赖 PG 中 `rag_embeddings` 是否有数据；无 embedding 时自动 keyword 降级。
6. **生产就绪度**：README 自述为“可跑原型，非成熟商业产品”；Bearer/Admin key、prod 校验在 `validate_startup.py` 中有要求。
7. **本地敏感配置**：`.env`、`frontend/.env.local`、`_local/data/cookies/` 为运行时敏感/私有数据，本包已排除；审查者无法直接复现需 key 的外部能力（LLM、DashScope ASR、腾讯 OCR 等） unless 自行配置。
